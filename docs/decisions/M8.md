# M8 — Design Decisions

## M8-01 — Feature flag

**Decision:** Gas Town requires `BDRALPH_GAS_TOWN=enabled` to activate. Off by default.
Any invocation without the flag returns a clear error with instructions.

**Rationale:** Gas Town is a high-risk, high-complexity feature. Explicit opt-in prevents
accidental activation. See M0-09.

---

## M8-02 — Loop declaration schema

**Decision:** The operator (or external copilot) declares the Gas Town session in JSON:

```json
{
  "gas_town": {
    "session_id": "gt-20240325-001",
    "loops": [
      {
        "id": "loop-a",
        "task": "Implement input validation in TaskService",
        "worktree": "gt-loop-a",
        "budget_fraction": 0.4
      },
      {
        "id": "loop-b",
        "task": "Add API endpoints for TaskService",
        "worktree": "gt-loop-b",
        "budget_fraction": 0.4
      }
    ],
    "integration_task": "Merge loop-a and loop-b changes into main branch",
    "budget_reserve_fraction": 0.2
  }
}
```

`budget_fraction` values across all loops plus `budget_reserve_fraction` must sum to 1.0.
The reserve covers the integration ralph and Second Mind coherence check.

**Rationale:** Declarative schema makes dependencies explicit and validates before
any loop fires. Second Mind validates the declaration before execution starts.

---

## M8-03 — Worktree lifecycle

**Decision:** Git worktrees are created before any loop fires, one per loop entry.
On successful integration, worktrees are removed. On failure (any loop crashes,
watchdog halts, or integration fails), worktrees persist for operator debug.

**Rationale:** Pre-creation ensures all loops start from a clean, known state.
Persistence on failure preserves the state needed to diagnose what went wrong.

---

## M8-04 — Watchdog behavior by trigger

**Decision:** The deterministic bash watchdog monitors all loops and acts as follows:

- **File conflict** (two loops touching the same file): pause the more recent loop,
  notify Second Mind. Second Mind decides whether to resume, redirect, or halt.
- **Crash** (loop process dies unexpectedly): mark loop as `failed`, continue remaining
  loops, notify Second Mind.
- **No progress** (N iterations without SHIP, where N = `--escalate-after` threshold):
  notify Second Mind. Loop continues while Second Mind evaluates.

**Rationale:** Graduated response — conflicts are recoverable (pause), crashes are
isolatable (continue others), stagnation is observable (notify without halting).

---

## M8-05 — Watchdog behavior is open to revision

**Decision:** M8-04 is the initial watchdog behavior. It is explicitly marked as open
to revision based on real usage experience.

**Condition to reopen:** After first real Gas Town execution, review watchdog trigger
thresholds and responses. Specific questions to revisit:
- Is "pause more recent loop" the right conflict resolution, or should it be
  "pause the loop with less progress"?
- Should N-iterations-without-progress halt the loop instead of just notifying?
- Should the watchdog have a maximum intervention count before halting everything?

**Rationale:** Gas Town is complex enough that initial behavior will need tuning.
Marking it explicitly avoids treating M8-04 as settled when it is not.
