# M7 — Design Decisions

## M7-01 — Native Gemini implementation

**Decision:** Native Gemini support is implemented as `src/loop/providers/gemini.ts`
using the `@google/generative-ai` SDK. `llm-delegate.sh` detects `WORKER_PROVIDER=google`
and delegates to this module via `npx tsx src/loop/providers/gemini.ts`.

**What "native" means:** no curl. The current implementation in `llm-delegate.sh` already
supports Gemini via curl (provider chain entries `gemini-flash`, `gemini`). M7 replaces
the curl path with the official SDK — more reliable, typed, and maintainable.

**Rationale:** The SDK handles auth, retries, and response parsing more robustly than
raw curl. TypeScript typing catches provider API changes at compile time.

---

## M7-02 — Provider chain defaults unchanged

**Decision:** M7 only changes the implementation (curl → SDK). Provider chain defaults
(`BDRALPH_L2_PROVIDER_CHAIN`, `BDRALPH_L3_PROVIDER_CHAIN`) remain unchanged. Operators
who want native Gemini as worker set `WORKER_PROVIDER=google`.

**Rationale:** Changing defaults would affect all existing deployments. The opt-in
pattern (`WORKER_PROVIDER=google`) is consistent with how other BDRALPH_ vars work.

---

## M7-03 — Cost guard Gemini pricing

**Decision:** Gemini prices are configurable via env vars:
- `BDRALPH_GEMINI_INPUT_PRICE` — USD per input token
- `BDRALPH_GEMINI_OUTPUT_PRICE` — USD per output token

Defaults are hardcoded at the time of the milestone release. Operators update the
env vars if Google changes pricing.

**Rationale:** Prices change independently of code. Env vars allow updates without
code changes. Hardcoded defaults work out of the box for most operators.
